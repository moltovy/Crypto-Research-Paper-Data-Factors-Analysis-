## 1. 🆕 New Features and Updates

### Cursor Automations

| Field | Details |
|-------|---------|
| **Announcement** | March 5, 2026 |
| **Version** | Cursor v2.6.x |
| **Status** | GA (General Availability) |

Cursor Automations enables IDE-based agents to work autonomously even when you're away from the computer. Agents now operate via **triggers** rather than just "prompt-and-monitor" interactions.

**Use Cases:**
- Auto code review when a PR is opened on GitHub
- Create JIRA task and write code from a Slack message
- Read logs and open a fix PR from PagerDuty incidents

**How to Use:**

Automations are configured via the Cursor Dashboard:

1. **Select Trigger:** GitHub PR (Push), Slack Webhook, Linear Issue, PagerDuty Incident, or Cron (Scheduled Task)
2. **Define Task:** Specify what the agent should do via `.cursorrules` or a custom command

**Example task definition:**
```
Run apidog test. If fails, create Linear ticket and ping #dev-alerts on Slack.
```

## 2. 🐛 Bug Reports and Workarounds

### Zombie Revert Bug — UPDATED

| Field | Details |
|-------|---------|
| **Severity** | Critical 🔴 |
| **First Report** | January 2026 |
| **Last Update** | March 2026 |
| **Affected Versions** | 2.4.x - 2.6.x |
| **Status** | 🔵 Workaround Available / Partially Fixed |

The Cursor team confirmed the root causes this month:

**1. Agent Review Conflict:**
Having the same file open in both the standard editor and "Agent Review" tab causes changes to be overwritten when "Fix in Chat" is used.
- ✅ **Fix:** Close the Agent Review tab before sending prompts.

**2. Cloud Sync Conflict:**
OneDrive, Google Drive, or iCloud syncing faster than Cursor's file locking mechanism reverts files.
- ✅ **Fix:** Exclude your project folder from cloud sync.

**3. Format On Save Conflict:**
Extensions like Prettier, Black, ESLint with Format on Save overwrite AI-written blocks.
- ✅ **Fix:** Use `cursor --disable-extensions` during AI-heavy work, or disable Format on Save.

### Linux (Ubuntu/GNOME) Crash Loop

| Field | Details |
|-------|---------|
| **Severity** | High 🟠 |
| **First Report** | March 10, 2026 |
| **Affected Version** | 2.6.x |
| **Platform** | Linux (Ubuntu 24.04 Wayland/GNOME) |

Desktop session crashes and loops back to login screen after authentication.

**Workaround:**
```bash
# Access via TTY terminal or recovery mode
rm -rf ~/.config/Cursor
```

A corrupted Cursor config/cache folder crashes GNOME. Deleting it provides a clean start.

## 6. 🔥 New Workflows

### 1. Context 7 MCP Server (Game Changer)

Pulls up-to-date documentation for 2000+ libraries into the LLM in real-time. Prevents agents from writing outdated version code (hallucinations).

**Setup (`.cursor/mcp.json`):**
```json
{
  "mcpServers": {
    "context7": {
      "command": "npx",
      "args": ["-y", "@upstash/context7-mcp"]
    }
  }
}
```

### 2. "Linear vs Looped" Workflow

Old-gen "Vibe Coding" → out. "Agentic Engineering" → in. (Andrej Karpathy)

- **Linear:** Quick prototype or simple bug fix → `Cmd+K` "Prompt → Code"
- **Looped:** Complex features → Agent loop: "Plan → Write Test → Code → Test → Read Error → Fix"

### 3. New .cursorrules Tips

**Anti-Hallucination Rule:**
```markdown
If you're about to import an npm package, first use the terminal agent to run
`npm list` and verify the package is installed in the project with 100% certainty.
```

**Interface Freeze Rule:**
```markdown
When adding a new feature, NEVER write implementation directly.
First define types/interfaces, wait for my approval, then write the function.
```
